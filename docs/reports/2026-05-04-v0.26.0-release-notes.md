# v0.26.0 release notes draft

Date: 2026-05-04
Predecessor: v0.25.7 (2026-04-26)
Status: **draft** — paste/edit into `git tag -a v0.26.0 -m "..."` when
ready to cut.

## Headline

Plugin authoring + provider/model overrides + cross-platform boot +
repo discovery hardening.

Five new top-level CLI flags, one new plugin subcommand, two new EPs,
19 inline call-sites consolidated into shared helpers, and the boot
path no longer breaks on Atomic Fedora's `/home → /var/home` symlink.

## What landed

### CLI

- `stado --version` — wires cobra's standard global flag to the same
  source as the existing `stado version` subcommand.
- `stado --provider <name>` / `stado --model <id>` — persistent root
  flags that override `[defaults].provider` / `[defaults].model` for
  one invocation. Honoured by the TUI, `stado run`, and subcommands
  inheriting the persistent flag.
- `stado plugin run --workdir <path>` — override the plugin's
  `host.Workdir`. Pass `--workdir=$PWD` when the plugin reads files
  from the operator's repo. Default unchanged (install dir) for
  backward compatibility. EP-0027.
- `stado plugin run --with-tool-host` — wires `host.ToolHost` so
  plugins importing bundled tools (`stado_http_get`,
  `stado_fs_tool_*`, `stado_lsp_*`, `stado_search_*`) can be
  exercised end-to-end from the CLI. Refuses plugins declaring
  `exec:bash` because no `sandbox.Runner` is available without the
  agent runtime, and EP-0005 forbids substituting human approval for
  runtime policy. EP-0028.
- `stado plugin gc [--keep N] [--apply]` — sweeps older installed
  plugin versions per (signer, name) group. Dry-run by default;
  trust-store and rollback pins deliberately untouched.

### Plugins / library

- `internal/workdirpath` exports four new helpers:
  - `LooksLikeRepoRoot(dir) bool` — predicate that distinguishes a
    real git working tree from an empty `.git/` artefact (the
    failure mode that fooled six inline parent-walks across the
    codebase). EP-0027.
  - `FindRepoRoot(start) string` / `FindRepoRootOrEmpty(start) string` —
    parent-walk variants matching the historical semantics callers
    relied on; both now share one tightened predicate.
  - `MkdirAllUnderUserConfig(path, perm) error` /
    `OpenRootUnderUserConfig(path) (*os.Root, error)` — anchor the
    no-symlink walk at the operator's HOME / XDG_*_HOME trust
    ancestors. Below the anchor, no-symlink enforcement still
    applies (defends against in-user-space attackers); above the
    anchor, OS-level symlinks are accepted (the operator's
    environment). EP-0028.
- 6 inline `findRepoRoot`/`findCurrentWorktreeRoot`/`findCurrentRepoRoot`
  walkers across `cmd/stado`, `internal/runtime`, `internal/memory`
  consolidated to delegate to the shared predicate.
- 13 HOME-rooted `MkdirAllNoSymlink` / `OpenRootNoSymlink` callers
  migrated to the trust-anchor variants (config dir, state dir,
  worktree dir, audit keys, memory store, plugin install / state
  files). The strict from-`/` versions stay in place for genuinely
  untrusted callers (in-repo plugin-sandbox writes).

### Inference

- `[inference.presets.<name>].api_key_env = "FOO_API_KEY"` plumbs a
  named env var through the OAI-compat client; previously only
  hardcoded preset names (litellm, groq, etc.) could pick up a key.
- `ollama-cloud` is now a builtin OAI-compat preset
  (`https://ollama.com/v1`, `OLLAMA_CLOUD_API_KEY`). No more
  litellm-aliasing workaround.
- Anthropic auto-raises `max_tokens` when thinking is enabled
  (`max_tokens > thinking.budget_tokens` is required by the
  provider; defaults previously violated this and surfaced as a
  400). Stado widens to `budget + 1024`, never lowering an explicit
  larger ceiling.

### Fixes

- `stado` no longer fails to boot on Fedora Atomic / Silverblue and
  similar setups where `/home` is a symlink to `/var/home`. The
  user-facing error was
  `config: create config dir: directory component is a symlink: home`.
  Fixed by the trust-anchor MkdirAll model above. EP-0028.
- Empty `/tmp/.git/` no longer fools session GC + lesson document
  paths. The naive `os.Stat(.git)` predicate accepted any `.git`
  entry, including stray empty dirs left by other tools. Fixed at
  the source by `LooksLikeRepoRoot`. Test impact: this fixed
  `TestSessionGC_ApplyActuallyDeletes` and four `TestLearningCLI_*`
  tests that had been failing on `main` in any CI/dev env with
  `/tmp/.git/` pollution. EP-0027.
- Warn when a stale `system-prompt.md` template drops
  `{{ .ProjectInstructions }}` — without this advisory, project
  rules in `AGENTS.md` / `CLAUDE.md` could silently fail to reach
  the model on installs predating the template's hook.

### Security pass (carried over from before v0.26.0 work began)

- Validated pinned plugin pubkeys against manifest fingerprints.
- Validated OAI-compatible endpoints (HTTPS, host present, no
  URL-embedded credentials).
- Hardened prompt and symbol reads through the no-symlink opener.
- Capped prompt, template, plugin install, and bundled-binary cache
  reads to bounded sizes.

## EPs added

- **EP-0027 — Repo-root discovery: single predicate, single helper.**
  Single source of truth for "what counts as a git working tree";
  consolidates 6 inline walkers behind one tightened predicate.
- **EP-0028 — `plugin run --with-tool-host` + HOME-rooted MkdirAll.**
  Two related improvements: wire `host.ToolHost` from CLI so plugins
  importing bundled tools work, and anchor HOME-rooted MkdirAll at
  XDG/HOME trust roots so cross-platform boot works.

## Test impact

- Full `go test ./...` green at HEAD.
- The two pre-existing test failures (`TestSessionGC_ApplyActuallyDeletes`,
  `TestLearningCLI_Document*`) are FIXED by EP-0027 — they had been
  failing silently on `main` for any environment with stray `/tmp/.git/`.
- New tests: `internal/workdirpath/repodisco_test.go`,
  `cmd/stado/plugin_run_tool_host_test.go`, `cmd/stado/plugin_gc_test.go`.

## Migration notes

- **No breaking changes.** All new CLI flags are additive opt-ins.
  `stado plugin run` default behaviour is unchanged unless `--workdir`
  or `--with-tool-host` is passed.
- **Behaviour change for stray `.git/` dirs.** Repo-root discovery
  now rejects empty `.git/` directories. Production impact: a user
  who ran `stado run` from `/tmp/myproject` (no real `.git`) used to
  get sessions silently pinned to `/tmp` if `/tmp/.git/` happened to
  exist. v0.26.0 pins to `/tmp/myproject` (the actual cwd) instead.
  Sessions already pinned to disk are unaffected (the user-repo pin
  in each worktree is read directly).
- **`MkdirAllNoSymlink` semantics.** Strict from-`/` callers (FS
  writes from inside the plugin sandbox) still get the strict check.
  HOME-rooted callers were migrated to `MkdirAllUnderUserConfig`,
  which accepts OS-level symlinks above HOME but enforces no-symlink
  below. Plugins that depend on the old behaviour at HOME-rooted
  paths shouldn't exist (the strict version was over-strict for that
  use); if any do, file an issue.
